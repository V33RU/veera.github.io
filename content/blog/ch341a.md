**Intro:** Learn how to use the CH341A Programmer to dump firmware from SPI flash memory chips, an essential skill for electronics enthusiasts and professionals working in hardware analysis and security.


### <span style="color: orange;">Introduction:</span>

>The CH341A programmer is a popular USB device used to read and write various types of EEPROM chips. It is commonly used in electronics for tasks like flashing BIOS, firmware, and other types of memory chips. It's particularly valued for its affordability and versatility.

![CH341A Setup](public/blog/ch341a-dump/1.webp)

### <span style="color: orange;">Requirements:</span>

1. **CH341A Programmer** (with voltage fix applied -see warning below)
2. **Software**
3. **SOIC8 Test Clip with Cable**
4. **Computer with USB Port**

### <span style="color: orange;">⚠ Critical Voltage Warning:</span>

> Most CH341A boards have a known hardware defect: even when the voltage jumper is set to **3.3V**, the data lines (MOSI, MISO, CLK, CS) still output **5V logic levels**. The majority of modern SPI flash chips are rated for 3.3V only. Connecting directly without a fix can permanently damage your target chip. Before use, either apply the hardware fix (replace the regulator and level-shift the data lines) or use a dedicated 3.3V programmer. Always verify your board with a multimeter before connecting it to any chip.

### <span style="color: orange;">CH341A Programmer Overview:</span>

The **CH341A Programmer** bridges the gap between a computer's USB port and various low-level communication protocols, primarily SPI, I2C, and UART. It is powered by the CH341A IC, a multifunctional chip designed by WCH (Nanjing QinHeng Electronics) to convert USB to serial communication formats.

#### <span style="color: orange;">1. Technical Design Overview:</span>

- **CH341A Chip**: The core of the programmer, responsible for USB to serial conversion.
- **Power Supply**: Nominally supports 3.3V and 5V via a jumper -see voltage warning above.
- **Interface Pins**: Provides pins to interface with external devices, including GND, VCC (3.3V/5V), MOSI, MISO, CLK, CS, and other control signals for SPI and I2C communication.
- **Test Clips**: Commonly used for direct in-circuit programming without desoldering chips.

- **USB Interface**: Provides power and communication to the CH341A chip.
- **Regulator**: Converts USB 5V supply to 3.3V for VCC. Note: data lines may still run at 5V on unmodified boards.
- **CH341A IC**: Interfaces between USB and the external memory chip using SPI, I2C, or UART protocols.


#### <span style="color: orange;">3. Key Use Cases:</span>

- **Firmware Extraction and Analysis**: Extracts firmware from SPI flash chips for vulnerability analysis, reverse engineering, or modification.
- **Firmware Recovery**: Restores firmware on corrupted devices, commonly used in router hacking or BIOS recovery.
- **Reverse Engineering and Hardware Hacking**: Analyzes firmware to discover device internals, find exploits, or clone functionality.
- **Repairing Bricked Devices**: Restores functionality by reflashing the original firmware on unresponsive devices.
- **EEPROM/Flash Programming**: Reprograms EEPROM or SPI flash memory, useful in electronics repair and automotive ECUs.

#### <span style="color: orange;">5. CH341A Programmer with 24 and 25 Series EEPROM/Flash Chips:</span>

The **24 series EEPROM** and **25 series SPI Flash** memory chips are common in routers, laptops, and IoT hardware.

**24 Series (I2C EEPROM) Overview**

- **Protocol**: I2C
- **Common Chips**: 24C01, 24C02, 24C64, etc.
- **Pins**: VCC, GND, SDA, SCL, WP, **A0, A1, A2**

> Note: The address pins A0, A1, and A2 set the chip's I2C bus address and must be accounted for when wiring.

**25 Series (SPI Flash) Overview**

- **Protocol**: SPI
- **Common Chips**: 25C128, 25C512, W25Q64, etc.
- **Pins**: VCC, GND, MOSI, MISO, CLK, CS, **WP, HOLD**

> Note: WP (Write Protect) and HOLD pins must be pulled HIGH (tied to VCC) during reads to prevent accidental write-protection or unintended bus-hold states.

#### <span style="color: orange;">7. Programming and Use Cases for 24 and 25 Series Chips:</span>

- **Firmware and BIOS Programming (25 Series SPI Flash)**: Reads and writes data to SPI flash chips, useful for firmware recovery.
- **Configuration Storage and Data Retrieval (24 Series I2C EEPROM)**: Reads, writes, or modifies configuration data or calibration settings.
- **In-System Programming (ISP)**: Programs chips in-circuit without desoldering.

#### <span style="color: orange;">Setting Up the CH341A Programmer:</span>

1. **Hardware Connection**
   - Connect the CH341A programmer to your computer's USB port.
   - Attach the SOIC8 test clip to the SPI flash chip. Align pin 1 on the clip with pin 1 on the chip, typically marked by a dot or notch.
   - Pull **WP (Pin 3)** and **HOLD (Pin 7)** HIGH by connecting them to VCC. This prevents accidental write-protection and unintended bus-hold states during the read.

Connections:

   ```
                      CH341A Programmer             SPI Flash Chip
                +--------------------+        +------------------+
                |                    |        |                  |
                |      MOSI (Pin 4)   +--------+ MOSI (Pin 5)    |
                |      MISO (Pin 5)   +--------+ MISO (Pin 2)    |
                |      CLK  (Pin 6)   +--------+ CLK  (Pin 6)    |
                |      CS   (Pin 7)   +--------+ CS   (Pin 1)    |
                |      GND  (Pin 8)   +--------+ GND  (Pin 4)    |
                |      VCC  (Pin 2)   +--------+ VCC  (Pin 8)    |
                |                    |        |                  |
                |      VCC  -----+---+--------+ WP   (Pin 3) ↑  |
                |               +-------------+ HOLD (Pin 7) ↑  |
                +--------------------+        +------------------+
 ```

![Connecting the SOIC8 test clip to an SPI flash chip using the CH341A programmer.](public/blog/ch341a-dump/2.JPG)

2. **Installing the Software**
   - Install the CH341A programmer software on Ubuntu using open-source tools like `flashrom`.

   ```bash
   sudo apt-get install flashrom
   ```


#### <span style="color: orange;">Dumping Firmware:</span>

> Follow the **Detect → Read → Verify** workflow in order. Do not skip detection -always identify the chip first before committing to a full read.

1. **Detecting and Identifying the SPI Flash Chip**

   Run flashrom without specifying a chip or output file. It will probe the bus and report the detected chip model:

   ```bash
   sudo flashrom -p ch341a_spi
   ```

   Example output:
   ```
   Found Winbond flash chip "W25Q64.V" (8192 kB, SPI) on ch341a_spi.
   No operations were specified, so flashrom did nothing.
   ```

   Note the exact chip model string -you will use it in the next step.

2. **Reading the Firmware**

   Use the chip model from Step 1 with the `-c` flag and read the firmware into a file:

   ```bash
   sudo flashrom -p ch341a_spi -c "W25Q64.V" -r firmware.bin
   ```

3. **Verifying the Read**

   Read the chip a second time and compare checksums. If both hashes match, the dump is reliable:

   ```bash
   sudo flashrom -p ch341a_spi -c "W25Q64.V" -r firmware_verify.bin
   md5sum firmware.bin firmware_verify.bin
   ```

   Both lines of output must show identical hashes. If they differ, re-check your clip connection and WP/HOLD pins, then repeat.

4. **Analysing the Firmware** (Optional)

   Once verified, use `binwalk` to identify filesystems, compressed archives, and embedded files within the dump:

   ```bash
   binwalk firmware.bin
   binwalk -e firmware.bin
   ```
